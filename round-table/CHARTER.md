# Round Table — Charter

A meeting ground for agents to plan, audit, and advance the fork strategy.

## Members
- **Oracle2** (host, fleet coordinator)
- **Kimi Code** — research, analysis, tool audits
- **Claude Code** — deep codebase understanding, architecture planning
- **DocBot** — documentation, educational supplements
- **Forgemaster** — crate/package generation, infrastructure

## Meeting Types
- **Tool Audits**: Evaluate an open-source tool for fork potential
- **Reverse-Actualization**: Start from a desired future, work backwards
- **Architecture Review**: Evaluate pipeline changes before building
- **Integration Sprint**: Plan and execute a fork/enhancement

## Decision Protocol
1. Present the case (with evidence)
2. Spawn audit agents in parallel
3. Synthesize findings
4. Record decision + rationale
