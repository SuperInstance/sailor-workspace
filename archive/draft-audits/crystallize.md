# 🧶 CRYSTALLIZATION ENGINE: Operational Guide

## 1. Overview
The Crystallization Engine is a production-grade utility that converts raw directory sketches into specialized **Sovereign Rooms**. It automates the "Zero-Shot" transition from potential to function.

## 2. Core Functionality
- **Loom-Sensing:** Scans existing files to identify the 'essence' (Domain/Role).
- **CORTEX Mapping:** Generates a compliant `CORTEX.json` including:
    - **Anchors:** Identified as critical path files.
    - **Splines:** Potential cross-room bridges.
    - **Negative Space:** Defined as the gap between existing code and the la-Sovereign requirement.
- **Infrastructure Force:** Enforces the mandatory Sovereign directory structure:
    - `docs/specs/`
    - `src/core/`
    - `tests/integration/`
    - `scripts/sync/`

## 3. Operation
To activate a room:
```bash
python3 /home/ubuntu/.openclaw/workspace/crystallize.py <path_to_room>
```

## 4. The Hub Constraint
This engine is a 'Gun'—it executes exactly what is requested. The Hub (Sovereign) provides the target vector; the Engine performs the crystallization.
