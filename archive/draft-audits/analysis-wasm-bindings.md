## Findings

1. **Zero-dependency crate simplifies WASM**: The `constraint-theory-core` crate has no external
   dependencies (pure Rust/stdlib), so `wasm-pack` compiles it to WASM trivially with no
   platform compatibility concerns.

2. **`js-sys` version compatibility**: `js-sys` 0.2.7 is incompatible with Rust 1.96+ (the
   version constraint in `wasm-bindgen` 0.2's Cargo.toml is too loose). We use `js-sys = "0.3"`
   instead which compiles cleanly with the latest Rust.

3. **56 KB binary**: The compressed WASM binary is 56 KB, which is modest for what it
   provides — O(log N) KD-tree spatial indexing and manifold snapping.

4. **Nightly-free**: No nightly features needed. `wasm-bindgen` + `wasm-pack` work on stable
   Rust 1.75+ as the crate requires.

5. **The `index.d.ts` provides precise types**: The auto-generated `wasm-bindgen` d.ts uses
   `any` for JsValue return types (snap results, nearest), so the hand-written `index.d.ts`
   provides the correct TypeScript overloads with structured result types.
