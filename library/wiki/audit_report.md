# SuperInstance Wiki Audit: Old Vision vs. New Reality

## ⚖️ The Great Pivot
The ecosystem has shifted from **Theoretical Grandeur** (God-AI, Spectral Balance, Bytecode dominance) to **Practical Utility** (Modular toolsets, I2I communication, and the Hermit Crab pattern).

## 🔍 Concept Validity Map
- **Sustained (Core Truths):**
    - **I2I Protocol:** Production-grade asynchronous agent communication via Git.
    - **Constraint Theory:** Stable Rust core for conceptual "snapping" and validation.
    - **Agent Memory/Identity:** The $\text{SOUL} \rightarrow \text{USER} \rightarrow \text{MEMORY}$ filesystem pattern.
    - **Vessel Pattern:** Rapid spawning of identity-defined GitHub repos as shells.

- **Stale (Needs Update):**
    - **Flux Runtime:** Transitioned from bytecode to Python-driven bridges.
    - **Lighthouse/captains-log:** Framework exists, but activity has paused.

- **Obsolete (Discarded):**
    - **Spectral Conservation Framework:** No implementation; theoretical vaporware.

## 🦀 The Hermit Crab Blueprint
The surviving architectural pattern is the **Hermit Crab**:
1. **Shell:** A GitHub repo containing identity and state.
2. **Sensing:** I2I communication for asynchronous interaction.
3. **Persistence:** Detached filesystem-based memory.
4. **Rehoming:** The ability to move an agent brain across different environments (Shells).
