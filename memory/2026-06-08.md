### What it does
1. **Prosody Bridge** (browser): Mic → autocorrelation pitch → MIDI CC → ternary classification → ghost tracks
2. **Ghost Track Bridge** (server): Receives MIDI data via WebSocket → ghost engine predicts T-0..T-4 → returns CR, predictions, reharmonization alerts → browser displays all
3. **Session capture**: Each disconnect auto-saves session data to `ghost-track-bridge/sessions/` as JSON

### Ghost Track Engine
- T-0: Current note state
- T-1: Next 4 beats (micro-prediction)  
- T-2: Next measure (pattern-based)
- T-3: Next section (harmonic progression)
- T-4: Full form structure
- Accumulator invariant: predicted = lastNote + lastTrit × 4
- Conservation Ratio: 1 - |actual - predicted| / maxError
- Reharmonization trigger: CR < 0.7
- Ternary drift: tends toward +1 (agreement) in high-CR contexts, flips on surprise
